Project 1: Hangman Game

Description

The hangman game is a digital replica of the classic hangman game. The program is a C-based hangman game that reads words from a file. Players guess letters to reveal a hidden word, with incorrect guesses counting toward a 6-strike limit. A special feature allows players to attempt guessing the full word after two consecutive correct letter guesses. The game runs round-by-round with an option to continue or quit after each word.

Skills Learned

  1. File I/O
    • Open a file for reading data
    • Read strings sequentially from a file
    • Handle file operations within a loop structure
    • Properly close files to prevent memory leaks
    • Work with external data sources to make programs more flexible and dynamic
  2. String Manipulation and Character Processing
    • Using strlen() to determine string lengths for array bounds
    • Implementing strcmp() for comparing strings
    • Creating and modifying C-style strings
    • Converting characters for case-insensitive comparison
    • Iterating through strings using both for loops and while loops with termination conditions
    • Properly terminating strings with '\0' after modification
  3. Array Management and Memory Handling
    • Declaring arrays with fixed sizes
    • Passing arrays to functions and understanding they decay to pointers
    • Using arrays to track game state (the letters guessed that is in the word and the total letters guessed)
    • Ensuring array bounds are respected (e.g., guessedLetters[len + 1] = '\0')
    • Distinguishing between character arrays and strings in C
  4. Function Design and Modular Programming
    • Creating clear function prototypes for better code organization
    • Designing functions with single, well-defined responsibilities
    • Using return values effectively to communicate results
    • Passing parameters appropriately (by value vs. by reference)
    • The importance of separating game logic into distinct functions (playRound, updateStarWord, etc.)
  5. Game State Management
    • Maintaining strike count and checking against maximum
    • Tracking consecutive correct guesses
    • Monitoring which letters have been guessed
    • Updating the masked word representation
    • Determining win/lose conditions
